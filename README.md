# The Northlander Wayfinder

An independent traveller's directory for the **Northlander train route**
between Toronto Union Station and Cochrane, Ontario. Restaurants, parks,
stays, attractions, and a community events calendar for every one of the
16 stops, plus a trip-planning companion app.

Two deployable surfaces, one repository:

- **`/site`** is `northlanderguide.com` (the Guide): a static, vanilla
  HTML/CSS/JS site served by Vercel from this folder.
- **`/northlander-app`** is `northlander.app` (the App): a SvelteKit PWA
  trip planner that loads listing + event data cross-origin from the
  Guide and stores everything else locally via Dexie.

Backend cron scripts under `/backend` are invoked by GitHub Actions
workflows in `.github/workflows`; there is no long-running server.

## Image pipeline

Every image under `site/images/` is committed to the repo as WebP at
quality 80. Hero photos that came in wider than 1600px were resized
during the one-time bandwidth pass; listing card photos stayed at the
800px Google Places returns. Going forward:

- `backend/sync-events.js` downloads Airtable attachments and transcodes
  to WebP via `backend/webp.js` (which shells out to `cwebp`). The
  events workflow installs the `webp` package on the runner before
  invoking the script.
- `backend/build-static.js` fetches Google Places photos for the
  listings grid and also transcodes to WebP. Run locally:
  ```
  brew install webp           # or sudo apt-get install -y webp
  node backend/build-static.js
  ```
  then commit the regenerated `site/images/listings/` and
  `site/listings-data.js`.
- `tools/convert-images-webp.mjs` is the one-shot converter for the
  initial pass. Re-run if a fresh batch of source JPGs ever lands.

## Working on it locally

Static site:

```
cd site
python3 -m http.server 8000
```

then open `http://localhost:8000`. The route map needs the server to
load `data.js`; opening the HTML file directly falls back to a list.

App:

```
cd northlander-app
npm install
npm run dev
```

then open `http://localhost:5173`.

Backend syncs run on push (GitHub Actions) so you rarely need to invoke
them locally. To test one manually you need the relevant secrets in
`backend/.env` (gitignored): `AIRTABLE_API_KEY`, `AIRTABLE_BASE_ID`,
`GOOGLE_PLACES_KEY` (when refreshing listings photos).

## Deploying

Both surfaces ship to Vercel. The Guide deploys from `/site` per
`vercel.json` at the repo root. The App deploys from `/northlander-app`
via the SvelteKit adapter configured in
`northlander-app/svelte.config.js`. Any push to `main` redeploys both.
